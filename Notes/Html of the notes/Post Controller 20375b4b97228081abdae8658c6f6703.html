Post Controller

Create Post : the route will change for this make a new route for Posts. Take content and description from the body. add the post id to the user and also the post count to the User’s db (using $addToSet and $inc). This will make it easy to access the posts from the user easily, store the ids of the post.

In post store the content and all and also the user id.

Update Post : To do this we must be sure who the doer is , it should only be the post owner, so check the current user with the owner of the post and take the description from the body and update the post.

Delete Post : This is same as the update post just be sure about the owner and remove the postid from the user’s db, use($pull, $inc with -1) for the removal of the particular post and also decreasing the post number.

Summery :

This page describes the core post controller functions:

  • Create Post:
    • Takes content and description from body
    • Adds post ID to user's database
    • Updates user's post count
    • Stores post content and user ID
  • Update Post:
    • Verifies post ownership
    • Updates post description
    • Only allows post owner to make changes
  • Delete Post:
    • Verifies post ownership
    • Removes post ID from user's database
    • Decrements user's post count
    • Deletes the post
